    pb_customization_rule (Resource)¶

    Example Usage¶

    resource "pb_customization_rule" "example" {
  name = "Block incognito browsing"
  mode = "active"

  controls = {
    incognito = {
      action = "block"
    }
  }

  # scope is optional. Omit it (or set is_any = true) to apply to everyone.
  scope = {
    users = {
      is_any = true
    }
  }
}

    Schema¶

    Required¶

    • controls (Attributes) Controls for customization rules. (see below for nested schema)
    • mode (String) The mode of the rule. The 'monitor' mode is not supported for this rule type.
    • name (String) The name or title of the rule.

    Optional¶

    • configuration_version (String) Configuration version to retrieve (draft, active, or version number)
    • description (String) The detailed description of the rule.
    • positioning (Attributes) Optional placement directive for the entity. On create, when omitted, the entity is placed at the top of the stack. On update, when omitted, the entity's current position is preserved. (see below for nested schema)
    • scope (Attributes) Provide scope to describe this rule (see below for nested schema)

    Read-Only¶

    • id (String) The unique identifier of the created resource (PULID, 29 chars).
    • metadata (Attributes) (see below for nested schema)
    • policy_profile (Attributes) The linked policy profile (see below for nested schema)
    • priority (Number) Order position of the rule in the list
    • section (Attributes) Section this rule belongs to (null if standalone) (see below for nested schema)

    Nested Schema for controls¶

    Optional:

    • admin_messages (Attributes) Control whether admin messages appear on the new tab page. (see below for nested schema)
    • auto_launch_external_applications (Attributes) Control whether websites can open external desktop apps without prompting users. (see below for nested schema)
    • automatic_client_certificate_selection (Attributes) Automatically selects matching client certificates for specified URLs to facilitate seamless authentication. (see below for nested schema)
    • background_image (Attributes) Set new tab page background (see below for nested schema)
    • brand_color (Attributes) Choose a brand color to be applied across all user-facing browser elements, including dialogs, block pages, and other elements (see below for nested schema)
    • browser_icon (Attributes) Set the icon for the browser in the operating system (see below for nested schema)
    • built_in_dns_client (Attributes) Choose whether DNS is resolved by the browsers built-in client or by the operating system. (see below for nested schema)
    • cached_session_duration (Attributes) Set the duration the browser can run using securely cached credentials without syncing to the Prisma Browser management console. (see below for nested schema)
    • company_logo (Attributes) Set the logo displayed in the browser on dialogs, homepage, and other pages. (see below for nested schema)
    • company_name (Attributes) Set the name displayed in the browser on dialogs, homepage, and other pages. (see below for nested schema)
    • custom_browser_preferences (Attributes) Apply custom browser preferences. Please contact Prisma Browser's support team for further guidance. When mode is enabled, preferences is required; when mode is disabled, preferences must not be set. (see below for nested schema)
    • custom_http_header (Attributes) Set a custom HTTP header to identify HTTP/S requests. (see below for nested schema)
    • custom_notice (Attributes) Set a customized notice that will be displayed to end-users when they use the Prisma mobile browser. (see below for nested schema)
    • custom_profile_preferences (Attributes) Apply custom browser profile preferences. Please contact Prisma Browser's support team for further guidance. When mode is enabled, preferences is required; when mode is disabled, preferences must not be set. (see below for nested schema)
    • custom_texts (Attributes) Customize and edit texts shown in browser dialogs and block pages. (see below for nested schema)
    • default_ai_provider (Attributes) Selected provider will appear as a "Prompt AI" shortcut in the New Tab Page Enterprise Omnibox. (see below for nested schema)
    • default_search_provider (Attributes) Control the default search engine provider for the address bar and context menu search access. (see below for nested schema)
    • deployment_upgrade (Attributes) Select Prisma Browser upgrade channel. (see below for nested schema)
    • desktop_sharing (Attributes) Control end-user ability to share their entire desktop screen or desktop apps during video conferencing sessions via web apps such as Zoom, MS Teams, and Slack. (see below for nested schema)
    • extension_force_install (Attributes) Set specific extensions to be installed automatically in Prisma Browser. (see below for nested schema)
    • force_page_refresh_on_policy_update (Attributes) Force a page refresh while the end-user is browsing to immediately enforce policy changes for certain controls. (see below for nested schema)
    • history_collection (Attributes) Collect new users' browsing history to identify trends and support enforcement decisions based on historical data. (see below for nested schema)
    • home_page (Attributes) Set the target page when a user clicks the Home Page button. (see below for nested schema)
    • identity_provider_synced_shortcuts (Attributes) Display a user's IDP synced company assigned applications as shortcuts on their New Tab page. (see below for nested schema)
    • incognito (Attributes) Control the ability to open Prisma Browser in Incognito mode. (see below for nested schema)
    • internal_network_detection (Attributes) Configure internal network detection to ensure traffic is routed in an optimal way. (see below for nested schema)
    • managed_shortcuts (Attributes) Set the applications that appear on the new tab page and in bookmarks. (see below for nested schema)
    • microsoft_auto_sso (Attributes) Enable automatic web-application user sign-in for Microsoft Entra ID accounts, based on the device signed-in account. (see below for nested schema)
    • new_tab_page (Attributes) Set the type of page when a new Prisma Browser or tab is opened. (see below for nested schema)
    • on_browser_startup (Attributes) Control the browser's 'On Startup' settings (see below for nested schema)
    • onboarding_wizard (Attributes) Configure the end-user onboarding wizard. (see below for nested schema)
    • page_translation_google_translate (Attributes) Control the Google Translation banner on foreign-language pages. (see below for nested schema)
    • prisma_browser_desktop_extender (Attributes) Control the use of Prisma Browser Local Extender (PABL). (see below for nested schema)
    • profile_sync (Attributes) Set up user profile sync across all the devices a user is logged in to. (see below for nested schema)
    • proxy_auto_configuration_pac_file (Attributes) Configure Prisma Browser to use a proxy auto-configuration (PAC) file. (see below for nested schema)
    • real_user_monitoring (Attributes) Control the use of RUM within Prisma Browser. (see below for nested schema)
    • request_desktop_mobile_site (Attributes) Override default site settings for specific domains on mobile devices. (see below for nested schema)
    • search_engine_content_filtering (Attributes) Enable content filtering for search engines to reduce exposure to inappropriate or distracting material (e.g., adult or violent content). This setting is supported by selected search engines, as appears in the list below. (see below for nested schema)
    • search_suggestions (Attributes) Show live search suggestions in the address bar as users type. (see below for nested schema)
    • suggest_default_browser (Attributes) Controls whether Prisma Browser suggests itself as the default browser to the user. (see below for nested schema)
    • theme_color (Attributes) Set Prisma Browser's theme color. (see below for nested schema)
    • traffic_flow (Attributes) Choose the way the Prisma Browser handles network traffic. (see below for nested schema)
    • user_agent (Attributes) Set additional User-Agent component to the default User-Agent string to identify HTTP/S requests. (see below for nested schema)

    Nested Schema for controls.admin_messages¶

    Required:

    • mode (String) Whether admin messages appear on the new tab page

    Optional:

    • messages (Attributes List) Custom admin messages appearing on the new tab page (see below for nested schema)

    Nested Schema for controls.admin_messages.messages¶

    Required:

    • description (String) Message body displayed on the new tab page
    • title (String) Message title displayed on the new tab page

    Optional:

    • external_link (String) Optional URL linked from the message. Must be provided together with externalLinkName.
    • external_link_name (String) Optional link name for the message URL. Must be provided together with externalLink.

    Nested Schema for controls.auto_launch_external_applications¶

    Required:

    • action (String) Whether matching websites can bypass the external application launch prompt

    Optional:

    • rules (Attributes List) Rules for auto-launching external applications (see below for nested schema)

    Nested Schema for controls.auto_launch_external_applications.rules¶

    Required:

    • allowed_origins (List of String) Origins allowed to launch the protocol without prompting
    • protocol (String) External application protocol scheme

    Nested Schema for controls.automatic_client_certificate_selection¶

    Required:

    • action (String) Whether automatic client certificate selection is enabled

    Optional:

    • rules_ (Attributes List) URL sets and certificate filters for automatic client certificate selection (see below for nested schema)

    Nested Schema for controls.automatic_client_certificate_selection.rules_¶

    Required:

    • filter (Attributes) Certificate matching criteria for the URL set (see below for nested schema)
    • url_patterns (List of String) URL patterns where matching client certificates are selected automatically

    Nested Schema for controls.automatic_client_certificate_selection.rules_.filter¶

    Optional:

    • issuer (Attributes) Optional issuer certificate criteria (see below for nested schema)
    • subject (Attributes) Optional subject certificate criteria (see below for nested schema)

    Nested Schema for controls.automatic_client_certificate_selection.rules_.filter.issuer¶

    Optional:

    • common_name (String) Issuer common name (CN)
    • location (String) Issuer location (L)
    • organization (String) Issuer organization (O)
    • organizational_unit (String) Issuer organizational unit (OU)

    Nested Schema for controls.automatic_client_certificate_selection.rules_.filter.subject¶

    Optional:

    • common_name (String) Subject common name (CN)
    • location (String) Subject location (L)
    • organization (String) Subject organization (O)
    • organizational_unit (String) Subject organizational unit (OU)

    Nested Schema for controls.background_image¶

    Required:

    • mode (String) New tab page background configuration

    Optional:

    • background_color (String) Custom fill color for the new tab background
    • background_image_filter_color (String) Custom overlay color applied on top of the background image
    • background_image_filter_mode (String) Whether to use the default image filter or set a custom filter overlay
    • background_image_filter_opacity (String) Opacity of the custom overlay applied on top of the background image
    • background_image_id (String) Background image asset identifier. Set backgroundImageId to the value returned by POST /assets/background-image.
    • invert_logo_color (Boolean) Whether to invert the logo color so it stands out on the homepage
    • page_mode (String) New tab page display mode

    Nested Schema for controls.brand_color¶

    Required:

    • mode (String) Whether to use the default brand color or a custom one

    Optional:

    • color (String) Brand color applied to user-facing browser elements

    Nested Schema for controls.browser_icon¶

    Required:

    • mode (String) Whether a custom browser icon is configured

    Optional:

    • browser_icon_id (String) Browser icon asset identifier. Set browserIconId to the value returned by POST /assets/browser-icon.

    Nested Schema for controls.built_in_dns_client¶

    Required:

    • action (String) How DNS resolution is handled

    Nested Schema for controls.cached_session_duration¶

    Required:

    • mode (String) Whether to use the default cached session duration or a custom duration

    Optional:

    • duration_days (Number) Maximum cached session duration in days

    Nested Schema for controls.company_logo¶

    Required:

    • mode (String) Whether a company logo is configured

    Optional:

    • company_logo_id (String) Company logo asset identifier. Set companyLogoId to the value returned by POST /assets/company-logo.

    Nested Schema for controls.company_name¶

    Required:

    • mode (String) Whether to display a company name or no name

    Optional:

    • name (String) Company name displayed in browser surfaces. Rejects CR, LF, and other control characters while allowing spaces and unicode.

    Nested Schema for controls.custom_browser_preferences¶

    Required:

    • mode (String) Whether custom browser preferences are applied

    Optional:

    • preferences (String) Custom browser preferences as a JSON object (e.g. '{"spellcheck.dictionaries":["en-US"]}'). Must be a JSON object; scalars, booleans, null, and arrays are rejected. Required when mode is enabled; must not be set when mode is disabled.

    Nested Schema for controls.custom_http_header¶

    Required:

    • mode (String) Whether to keep default headers or add a custom header.

    Optional:

    • custom_header_name (String) HTTP header name. Must contain only alphanumeric characters, underscores, and hyphens (mirrors the dashboard wizard charset).
    • custom_header_value (String) HTTP header value. Allowed characters mirror the dashboard wizard's punctuation allowlist; CR, LF, and other control characters are rejected.

    Nested Schema for controls.custom_notice¶

    Required:

    • action (String) Whether the custom notice is shown.

    Optional:

    • description (String) Notice description.
    • frequency (String) How often the notice is shown.
    • title (String) Notice title.

    Nested Schema for controls.custom_profile_preferences¶

    Required:

    • mode (String) Whether custom profile preferences are applied

    Optional:

    • preferences (String) Custom profile preferences as a JSON object (e.g. '{"homepage":"https://example.com"}'). Must be a JSON object; scalars, booleans, null, and arrays are rejected. Required when mode is enabled; must not be set when mode is disabled.

    Nested Schema for controls.custom_texts¶

    Required:

    • mode (String) Whether browser text overrides are configured

    Optional:

    • overrides (Attributes List) List of trigger-specific text overrides. Required when mode is customizeTexts. Must not be set when mode is useDefaultTexts. (see below for nested schema)

    Nested Schema for controls.custom_texts.overrides¶

    Required:

    • trigger (String) Trigger whose default text is overridden

    Optional:

    • description (String) Replacement body text for the selected trigger
    • link (Attributes) Optional external link shown with the custom text (see below for nested schema)
    • title (String) Replacement title for the selected trigger

    Nested Schema for controls.custom_texts.overrides.link¶

    Required:

    • label (String) Link text displayed to the end user
    • url (String) Link destination URL

    Nested Schema for controls.default_ai_provider¶

    Required:

    • action (String) Whether the default AI provider shortcut is shown.

    Optional:

    • provider (String) Default AI provider shown as the "Prompt AI" shortcut.

    Nested Schema for controls.default_search_provider¶

    Required:

    • action (String) Whether the default search provider is managed.

    Optional:

    • provider (String) Search provider selection.
    • provider_name (String) Default search provider display name.
    • recommended (Boolean) Whether this setting is recommended and user-changeable.
    • search_url (String) Default search provider URL template. Must be an http(s) URL (or a "{google:baseURL}" built-in template) containing the {searchTerms} placeholder.

    Nested Schema for controls.deployment_upgrade¶

    Required:

    • mode (String) Prisma Browser upgrade channel

    Optional:

    • expedite_critical_security_updates (Boolean) Whether critical security updates can bypass the configured delay. Applies when mode is default and versionUpdateDelayDays > 0, or mode is lts and patchDelayDays > 0. Defaults to true when applicable. Must not be set when the relevant delay is 0, or when mode is pinned or noUpdates.
    • force_rollback (Attributes) Whether to force rollback when the pinned version is older than latest. Applies when mode is pinned. Defaults to false for both platforms when omitted. (see below for nested schema)
    • grace_period_hours (Number) Time in hours users can defer the forced update relaunch. Applies when mode is default, lts, or pinned. Defaults to 4 when omitted. Must not be set when mode is noUpdates.
    • major_release_delay_days (Number) Delay in days before receiving the next available LTS major release. Applies when mode is lts.
    • patch_delay_days (Number) Delay in days before receiving LTS security and bug patches. Applies when mode is lts. Must be less than or equal to majorReleaseDelayDays.
    • pinned_versions (Attributes) Pinned target versions by desktop OS. Required when mode is pinned. Must not be set for other modes. (see below for nested schema)
    • version_update_delay_days (Number) Delay in days before receiving the latest default-channel version. Applies when mode is default.

    Nested Schema for controls.deployment_upgrade.force_rollback¶

    Optional:

    • mac (Boolean) Whether to force macOS instances to roll back to the pinned version
    • windows (Boolean) Whether to force Windows instances to roll back to the pinned version

    Nested Schema for controls.deployment_upgrade.pinned_versions¶

    Required:

    • mac (String) Pinned macOS browser version
    • windows (String) Pinned Windows browser version

    Nested Schema for controls.desktop_sharing¶

    Required:

    • action (String) The action for this control.

    Nested Schema for controls.extension_force_install¶

    Required:

    • extensions (Attributes List) Extensions installed automatically in Prisma Browser (see below for nested schema)

    Nested Schema for controls.extension_force_install.extensions¶

    Required:

    • id (String) Chrome extension ID (32 lowercase letters)
    • source (String) Extension source. When customURL, sourceUrl is required and must not be set for chromeWebstore.

    Optional:

    • custom_configuration (String) Custom extension configuration JSON
    • install_in_incognito (Boolean) Whether automatic installation extends to Incognito mode
    • pinned (Boolean) Whether the extension is pinned to the toolbar
    • source_url (String) Custom extension package URL. Required when source is customURL. Must not be set when source is chromeWebstore.

    Nested Schema for controls.force_page_refresh_on_policy_update¶

    Required:

    • action (String) The action for this control.

    Nested Schema for controls.history_collection¶

    Required:

    • action (String) The action for this control.

    Nested Schema for controls.home_page¶

    Required:

    • mode (String) Home Page button mode.

    Optional:

    • custom_url (String) Custom Home Page URL. An http(s) URL or a schemeless host.
    • recommended (Boolean) Whether to make this Home Page setting recommended and user-changeable.
    • show_home_page_button (Boolean) Whether to always show the Home Page button.

    Nested Schema for controls.identity_provider_synced_shortcuts¶

    Required:

    • action (String) Whether IDP-synced shortcuts are displayed.
