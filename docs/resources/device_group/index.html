    pb_device_group (Resource)

    Example Usage

    resource "pb_device_group" "example" {
  name     = "Corporate macOS"
  platform = "macos"
}

    Argument Reference

    The following arguments are required:

    • name (String) Device group name
    • platform (String) Device group platform

    The following arguments are optional:

    • attributes (Attributes) (see below for nested schema)
    • configuration_version (String) Configuration version to retrieve (draft, active, or version number)

    Attributes Reference

    In addition to the arguments above, the following attributes are exported:

    • created_at (String) Created at timestamp
    • created_by (String) Created by user
    • device_group_id (String, Deprecated) Deprecated: use id instead. The ID of the newly created device group.
    • devices (List of String) Device IDs in this group
    • id (String) The ID of the newly created device group
    • updated_at (String) Updated at timestamp
    • updated_by (String) Updated by user

    Nested Schema for attributes

    Optional:

    • browser_brand (Attributes) Check if the device has specific browser brands and versions installed (see below for nested schema)
    • browser_eol (Attributes) Check if the device has browser versions that are end-of-life (see below for nested schema)
    • client_certificate (Attributes) Check if the device's client certificate is signed by the provided issuer certificate (see below for nested schema)
    • cs_zta_score (Attributes) Check if the device meets minimum CrowdStrike Zero Trust Assessment score requirements. If multiple score types are provided (basicScore, overallScore, breakdownScores), the latest one will be used. (see below for nested schema)
    • device_management (Attributes) Check if the device is managed by specific device management systems (e.g., Microsoft Intune, Jamf, Active Directory) (see below for nested schema)
    • device_manufacturer (Attributes) Check if the device is from specific manufacturers (e.g., Dell, HP, Lenovo) (see below for nested schema)
    • device_type (Attributes) Check if the device matches specific device types (e.g., desktop, laptop, virtual machine) (see below for nested schema)
    • disk_encryption (Attributes) Check if the device has disk encryption software installed and running (see below for nested schema)
    • endpoint_protection (Attributes) Check if the device has endpoint protection software installed and running (see below for nested schema)
    • file_existence (Attributes) Check if the device has all of the specified files present (see below for nested schema)
    • firewall (Attributes) Check if the device has firewall protection installed and running (see below for nested schema)
    • location_services (Attributes) Check if the device's location services can be accessed by Prisma Browser (see below for nested schema)
    • mobile_device_management (Attributes) Check if the mobile device is managed by specific mobile device management systems (see below for nested schema)
    • mobile_device_manufacturers (Attributes) Check if the mobile device is from specific manufacturers (e.g., Apple, Samsung) (see below for nested schema)
    • mobile_device_type (Attributes) Check if the mobile device matches specific types (e.g., phone, tablet) (see below for nested schema)
    • mobile_os_version (Attributes) Check if the mobile device is running a specific iOS or Android version (see below for nested schema)
    • mobile_root_jail_break_status (Attributes) Check if the mobile device has been rooted (Android) or jailbroken (iOS) (see below for nested schema)
    • mobile_screen_lock (Attributes) Check if the mobile device has screen lock protection enabled (see below for nested schema)
    • normal_osboot_mode (Attributes) Check if a device is running in OS normal boot mode (not safe mode, recovery mode or a pre-installation environment) (see below for nested schema)
    • os_password (Attributes) Check if the device has an OS authentication password configured with specific requirements (see below for nested schema)
    • os_version (Attributes) Check if the device is running a specific operating system version (see below for nested schema)
    • privileged_process (Attributes) Include only devices on which Prisma Browser is running with elevated/root permissions (see below for nested schema)
    • registry (Attributes) Check if the device has all of the specified registry key configurations (Windows only) (see below for nested schema)
    • remote_connection (Attributes) Check if the device has an active remote connection (RDP, Citrix ICA, etc.) (see below for nested schema)
    • running_processes (Attributes) Check if the device has all of the specified processes running (see below for nested schema)
    • screen_lock (Attributes) Check if the device has automatic screen lock enabled (see below for nested schema)
    • serial_number (Attributes) Check if the device's serial number is included in the provided list (see below for nested schema)
    • system_integrity (Attributes) Check if the device has advanced system integrity protection enabled (see below for nested schema)
