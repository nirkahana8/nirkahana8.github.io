    pb_security_rule (Resource)

    Example Usage

    resource "pb_security_rule" "example" {
  name = "Default security rule"
  mode = "active"

  controls = {
    # Security controls for the rule (see the Schema section below).
  }
}

    Schema

    Required

    • controls (Attributes) Controls for security rules. (see below for nested schema)
    • mode (String) The mode of the rule. The 'monitor' mode is not supported for this rule type.
    • name (String) The name or title of the rule.

    Optional

    • configuration_version (String) Configuration version to retrieve (draft, active, or version number)
    • description (String) The detailed description of the rule.
    • positioning (Attributes) Optional placement directive for the entity. On create, when omitted, the entity is placed at the top of the stack. On update, when omitted, the entity's current position is preserved. (see below for nested schema)
    • scope (Attributes) Provide scope to describe this rule (see below for nested schema)

    Read-Only

    • id (String) The unique identifier of the created resource (PULID, 29 chars).
    • metadata (Attributes) (see below for nested schema)
    • policy_profile (Attributes) The linked policy profile (see below for nested schema)
    • priority (Number) Order position of the rule in the list
    • section (Attributes) Section this rule belongs to (null if standalone) (see below for nested schema)

    Nested Schema for controls

    Optional:

    • advanced_browser_protection (Attributes) Stops zero-day WebAssembly attacks in real time by monitoring in-browser memory behavior and terminating malicious execution before any impact occurs. (see below for nested schema)
    • allowed_or_blocked_extensions (Attributes) Control browser extension availability to users based on extension ID or risk score. (see below for nested schema)
    • allowed_printers (Attributes) Control which printers can be used when printing from Prisma Browser. (see below for nested schema)
    • authentication_server_allowlist (Attributes) List the servers allowed to use Integrated Authentication. (see below for nested schema)
    • autofill_of_credit_cards (Attributes) Control saving credit cards in the browser. (see below for nested schema)
    • autofill_of_forms (Attributes) Control saving forms data in the browser. (see below for nested schema)
    • basic_authentication_over_http (Attributes) Set whether basic authentication can be used over HTTP websites. (see below for nested schema)
    • block_extensions_by_permissions (Attributes) Prevent users from running extensions that require certain permissions. (see below for nested schema)
    • browser_history (Attributes) Control the ability to delete history from the browser. (see below for nested schema)
    • browser_lock (Attributes) Require the user to unlock their browser. (see below for nested schema)
    • browser_self_protection (Attributes) Enables a kernel-mode driver that provides advanced runtime security for the browser. This protection is available only on Windows and applies only to devices where Prisma Browser is installed with admin permissions and the user is running the browser as admin. (see below for nested schema)
    • cast (Attributes) Control the ability to cast a tab or the whole desktop via Prisma Browser. Prisma Browser restart required to apply. (see below for nested schema)
    • concurrent_number_of_devices (Attributes) Control the maximum number of devices users can be logged into at the same time. Which fields are required or allowed depends on action and limitMode (see individual field descriptions). (see below for nested schema)
    • cookies (Attributes) Control the ability to store cookies on the browser. (see below for nested schema)
    • cookies_protection (Attributes) Control the encryption of stored cookies on the device. (see below for nested schema)
    • developer_tools_for_extensions (Attributes) Control DevTools to avoid inspection of browser extensions. (see below for nested schema)
    • dns_over_https (Attributes) Set DNS resolving on top of the HTTPS protocol, for encrypting the requests and their resolutions. resolverUrl and adnsEnabled=true are mutually exclusive. (see below for nested schema)
    • end_process_via_task_manager (Attributes) Choose whether end users can end browser processes via the browser task manager. (see below for nested schema)
    • enhanced_tracking_protection (Attributes) Manage tracking protection and cross-site tracking. (see below for nested schema)
    • file_system_api (Attributes) Control the use of the File System API, which might be exploited. (see below for nested schema)
    • flush_browser_data (Attributes) Set temporary browser sessions, so browser data will be cleaned upon close or time period. (see below for nested schema)
    • force_https (Attributes) Force using HTTPS instead of HTTP to reduce the risk of MitM attacks and sending sensitive information in cleartext. (see below for nested schema)
    • google_cloud_print (Attributes) Control the use of the Google Cloud Print, which might be exploited. (see below for nested schema)
    • hide_sensitive_data_from_extensions (Attributes) Prevent all extensions from accessing cookies and authorization data sent by the browser. (see below for nested schema)
    • internet_explorer_compatibility_mode (Attributes) Allow access to legacy websites using Internet Explorer compatibility mode. (see below for nested schema)
    • java_script_running_from_omnibox (Attributes) Control the ability to execute JavaScript code from the browser omnibox. (see below for nested schema)
    • java_script_v8jit_and_web_assembly (Attributes) Block JavaScript v8 JIT to reduce exploitation risks and to activate vulnerability mitigation techniques. Block WebAssembly (WASM) to reduce exploitation risks. (see below for nested schema)
    • kerberos_delegation_allowlist (Attributes) List the hosts that may forward a user's Kerberos ticket to downstream services. (see below for nested schema)
    • keylogging_protection (Attributes) Prevent keyloggers from capturing user input while using the browser (Windows only). (see below for nested schema)
    • launching_external_applications (Attributes) Configure the default behavior when users click links that launch external applications. You can then define exceptions for specific applications. (see below for nested schema)
    • legacy_password_manager (Attributes) Enable the Prisma Access Browser legacy password manager for managing and securing company passwords and secrets. (see below for nested schema)
    • local_fonts (Attributes) Control the access to local fonts installed on the device, which might be exploited. (see below for nested schema)
    • local_network_access_restrictions (Attributes) Manage website access to local network endpoints. (see below for nested schema)
    • mobile_password_saving (Attributes) Control passwords saving in the mobile browser. (see below for nested schema)
    • native_messaging_hosts (Attributes) Control which native messaging hosts can communicate with Prisma Browser and its extensions. (see below for nested schema)
    • notifications (Attributes) Control the ability to display notifications in the browser. (see below for nested schema)
    • open_links_in_external_apps (Attributes) Control the ability of other apps to open links from Prisma Browser. (see below for nested schema)
    • pages_with_insecure_content (Attributes) Set whether pages with insecure content are available. (see below for nested schema)
    • pages_with_ssl_errors (Attributes) Block "Proceed to [FQDN] (unsafe)" option when getting an SSL error. Such errors might appear during SSL MitM attack. (see below for nested schema)
    • password_manager (Attributes) Enterprise password management powered by Idira. (see below for nested schema)
    • pdfium (Attributes) Control the use of the PDFium library to render PDF pages, which might be exploited. When PDFium is disabled, encrypted PDF files won't be loaded in Prisma Browser. (see below for nested schema)
    • popups (Attributes) Control the ability to display popups in the browser. (see below for nested schema)
    • post_quantum_key_security (Attributes) Control whether to offer a post-quantum key agreement algorithm in TLS. (see below for nested schema)
    • print_preview (Attributes) Control the preview in the printing window, which might be exploited. (see below for nested schema)
    • quic_protocol (Attributes) Control the use of the QUIC protocol, which might be exploited. (see below for nested schema)
    • remote_debugging (Attributes) Control remote debugging on Android and iOS devices to prevent data leaks and simplify troubleshooting. (see below for nested schema)
    • remote_host_firewall_traversal (Attributes) Manage whether Remote Desktop may connect from outside the LAN or VPN by traversing firewalls. (see below for nested schema)
    • restrict_extension_host_permissions (Attributes) Prevent extensions from running scripts and accessing content in websites. (see below for nested schema)
    • sensors_api (Attributes) Control the use of the Sensor API, which might be exploited. (see below for nested schema)
    • session_refresh (Attributes) Periodically require the user to re-authenticate. (see below for nested schema)
    • strict_origin_isolation (Attributes) Isolates every origin in its own agent-cluster, preventing scripts on one sub-domain from interacting with another. (see below for nested schema)
    • third_party_cookies (Attributes) Set whether elements in the web page that aren't from the domain that's in the browser's address bar can set cookies. (see below for nested schema)
    • trusted_certificate_authorities (Attributes) Choose the certificate authorities trusted by Prisma Browser. (see below for nested schema)
    • user_data_directory_protection (Attributes) Enforce hardware based detections to prevent data directory and browser session theft. (see below for nested schema)
    • web_bluetooth_api (Attributes) Control the use of the WebBluetooth API, which might be exploited. (see below for nested schema)
    • web_clipboard_api (Attributes) Control the use of the Clipboard API, which might be exploited. (see below for nested schema)
    • web_gl_api (Attributes) Control the use of the WebGL library, which might be exploited. (see below for nested schema)
    • web_hid_api (Attributes) Control the availability of the WebHID API, which might be exploited. (see below for nested schema)
    • web_rtc (Attributes) Control the use of WebRTC mechanism, which might be exploited. Disabling it might impact the usability of websites working with WebRTC functionalities, like call/video streaming. (see below for nested schema)
    • web_serial_api (Attributes) Control the use of the WebSerial API, which might be exploited. (see below for nested schema)
    • web_usb_api (Attributes) Control the use of the WebUSB API, which might be exploited. (see below for nested schema)

    Nested Schema for controls.concurrent_number_of_devices

    Required:

    • action (String) Whether to limit the number of concurrent devices. When 'unlimited', no other fields may be set.

    Optional:

    • limit_mode (String) Whether the device limit applies to all devices combined or per device type. Required when action is 'limited'. Must not be set when action is 'unlimited'.
    • max_desktop_devices (Number) Maximum number of desktop devices (macOS/Windows/Linux). Use null for no limit, or 0 to block desktop devices entirely. Required when limitMode is 'byDeviceType'. Must not be set when limitMode is 'byTotalDevices' or when action is 'unlimited'. maxDesktopDevices and maxMobileDevices cannot both be null, and cannot both be 0.
    • max_mobile_devices (Number) Maximum number of mobile devices (iOS/Android). Use null for no limit, or 0 to block mobile devices entirely. Required when limitMode is 'byDeviceType'. Must not be set when limitMode is 'byTotalDevices' or when action is 'unlimited'. maxDesktopDevices and maxMobileDevices cannot both be null, and cannot both be 0.
    • max_total_devices (Number) Maximum total number of devices a user can be logged into across all device types. Required when limitMode is 'byTotalDevices'. Must not be set when limitMode is 'byDeviceType' or when action is 'unlimited'.
